# StudyHub v1.7.0 -- What's New

Hey StudyHub community,

We have been hard at work improving StudyHub and we are excited to share everything that has landed in v1.7.0. This is our biggest update yet -- focused on security, reliability, messaging improvements, and a ton of polish across the platform.

---

## Messaging Overhaul (StudyHub Connect)

This was one of our most requested areas for improvement, and we went deep.

**Unread notifications now actually work.** Previously, if you opened a conversation and read your messages, the unread badge could stick around after a page refresh. This has been completely fixed. Whether your real-time connection is active or not, reading a conversation always clears the notification -- on the messages page, in the quick-chat panel, and everywhere else.

**New unread badge on the navbar.** You will now see a red badge on the messages icon in the top navigation showing your total unread message count across all conversations. No more guessing whether someone messaged you -- you will know at a glance from any page.

**Unread conversations are visually distinct.** Conversations with unread messages now show bolder names and darker preview text so you can immediately spot what needs your attention.

**Typing indicators fixed.** The typing indicator no longer leaks across conversations. If someone is typing in one chat, it will not show up in a different one.

**Better error handling.** If the real-time connection drops, you will see a clear status message and everything continues working via standard requests. Messages will update on refresh.

---

## Study Groups Improvements

**Members tab fixed.** The Members tab on study group detail pages was showing "No Members" even when the group had members. This has been resolved -- member lists now load properly when you switch to the tab.

**Error retry.** If the study groups list fails to load, you now get a Retry button instead of just an error message sitting there with no way to recover.

---

## Study Sheet Activity Feed Fixed

The Activity section on study sheets was showing "Server error" for everyone. This was a backend bug that has been fixed. You can now see the full activity timeline for any sheet -- commits, contributions, and comments.

---

## Security Hardening

We did a comprehensive security audit and fixed every finding:

- All user-submitted content is now sanitized with a professional-grade library to prevent cross-site scripting attacks.
- Real-time messaging verifies room membership before broadcasting, so messages only go to actual participants.
- Search queries are length-capped and rate-limited to prevent abuse.
- PDF export is protected against injection attacks.
- Attachment scanning (ClamAV) is now mandatory in production -- no uploads bypass malware scanning.
- A global error handler catches unexpected server errors gracefully instead of exposing internal details.
- All API endpoints serving public data now have read rate limiters.

---

## Accessibility Improvements

- Notes list items are now proper interactive elements with full keyboard support. You can navigate your notes with Tab, Enter, and Space.
- Messaging modals (new conversation, delete confirmation) now trap focus correctly so keyboard and screen reader users stay oriented.
- Action buttons across messaging (reply, edit, delete) now have descriptive labels for screen readers.
- The conversation options menu is now a semantic button with proper ARIA attributes.

---

## Dark Mode Polish

We audited the entire frontend and replaced 50+ hardcoded color values with theme-aware tokens. Pages that previously looked washed out or had poor contrast in dark mode -- like Announcements, Email Suppressions, Courses, User Profile, and Contributions -- now look clean and consistent.

---

## Test Coverage

We added 22 new end-to-end tests covering:

- Messaging flows (conversation loading, empty states, message threads, DM auto-start, new conversation modal)
- Notification system (badge display, read/unread state, mark-all-read, deletion, navigation)

This means we catch regressions before they reach you.
