# StudyHub v1.5.0 Release Notes

**Release Date:** 2026-03-19
**Codename:** Clean Academic Pro

---

## Overview

StudyHub v1.5.0 is a major release delivering 6 weeks of planned improvements across security, collaboration, performance, and accessibility. This release transforms StudyHub from a study sheet sharing platform into a full-featured academic collaboration hub with version control, content moderation, and enterprise-grade security.

---

## New Features

### Sheet Lab — Version Control for Study Sheets
- Full commit history for every study sheet at `/sheets/:id/lab`
- Create snapshots with messages, restore to any previous version
- Side-by-side diff comparison between any two commits
- SHA-256 checksums for content integrity verification
- Anime.js-powered timeline UI with staggered animations

### Provenance Manifest System
- AES-256-GCM encrypted origin tokens prove sheet authorship
- Auto-generated on sheet creation and updates
- Admin-only decryption for dispute resolution
- Tamper detection compares current content hash against encrypted manifest
- Architecture ready for scheduled batch tamper scanning

### Feature Flags (OpenFeature-Compatible)
- Database-backed feature flags with admin CRUD
- Percentage-based rollout using deterministic user hashing
- JSON conditions for role-based and user-targeted flags
- In-memory cache with 30-second TTL for performance
- Express middleware (`req.flags.isEnabled()`) for backend gating
- React hook (`useFeatureFlag()`) for frontend gating

### Full-Text Search
- PostgreSQL GIN indexes on sheet title, description, and content
- Combined tsvector index for multi-field ranked search
- `ts_rank` scoring for relevance-ordered results
- Opt-in via `?fts=true` query parameter (gradual migration)
- Indexes also added for User (username) and Course (name, code)

### PWA Offline Support
- Service worker with network-first API caching and cache-first static assets
- Web app manifest for installability (standalone display)
- Apple mobile web app meta tags
- SVG and PNG icon support

### WebAuthn Passkeys (Admin Users)
- FIDO2 passkey registration and authentication for admin accounts
- Built with Node.js crypto — no external WebAuthn dependencies
- Manual CBOR decoder and DER signature verification
- ES256 and RS256 algorithm support
- Passkey management UI in Settings SecurityTab

### Client-Side Image Safety
- Canvas-based pre-upload image safety screening
- File size, dimension, and skin-tone heuristic checks
- Integration hook for future TF.js/NSFWJS model loading
- Non-blocking warnings (server-side OpenAI moderation remains authoritative)

### CI Quality Gates
- GitHub Actions workflow with lint, test, build, accessibility, and bundle size jobs
- Lighthouse CI config with performance (0.8), accessibility (0.9), and Core Web Vitals budgets
- Accessibility config with WCAG 2.2 AA rules and zero-tolerance violations
- 5 MB bundle size budget check

## Enhancements

### Upload Hardening
- Magic-byte validation on all file uploads (avatars, sheet attachments, post attachments)
- Triple validation: MIME type + file extension + magic bytes
- ZIP-based Office format detection (DOCX/XLSX/PPTX)
- Automatic file deletion on signature mismatch

### Admin Moderation Dashboard
- Stats overview: total users, new this week, published/draft sheets, feed posts
- Moderation overview: pending cases, active strikes, pending appeals (color-coded)
- Activity log: timeline of recent moderation actions with reviewer info
- Case detail view: full content, confidence score, evidence, quick action buttons
- Sort by date or confidence score

### Performance Optimization
- Web Vitals monitoring (LCP, INP, CLS) with PostHog integration
- Vite manual chunk splitting (react-vendor, animation)
- CSS code splitting enabled
- CLS prevention: min-height on root, aspect-ratio on avatars, contain on card grids
- Resource preconnect hints for Google Fonts and CDNs

### Accessibility
- Skip-to-content link on every page
- `aria-live` route change announcements for screen readers
- ARIA labels on all interactive elements (nav, search, modals, dialogs, buttons)
- `role="dialog"` and `aria-modal` on all modals and dialogs
- `id="main-content"` landmark on all pages
- Global `focus-visible` outline styles
- `prefers-reduced-motion` support (disables all animations)
- Print styles (hide nav, sidebar, toasts)
- Custom scrollbar styles
- `::selection` theming for light and dark modes
- Keyboard shortcut: Ctrl/Cmd+K for quick search

## Known Limitations

- WebAuthn uses a minimal CBOR/DER implementation; full FIDO2 compliance would benefit from `@simplewebauthn/server`
- TF.js NSFW model is stubbed — current safety check uses canvas heuristics; ML model loading is a future enhancement
- Full-text search uses `?fts=true` opt-in; production migration should switch the default after index verification
- PWA icons are SVG-based placeholders; production icons should be generated as PNGs at 192x192 and 512x512
- Challenge store for WebAuthn uses in-memory Map; production should use Redis or database-backed sessions
