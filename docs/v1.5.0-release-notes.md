# StudyHub v1.5.0 Release Notes

**Release Date:** 2026-03-19
**Codename:** Clean Academic Pro

---

## Overview

StudyHub v1.5.0 is a major release delivering 6 weeks of planned improvements across security, collaboration, performance, and accessibility. This release transforms StudyHub from a study sheet sharing platform into a full-featured academic collaboration hub with version control, content moderation, and enterprise-grade security.

---

## New Features

### Sheet Lab — Version Control for Study Sheets
- Full commit history for every study sheet at `/sheets/:id/lab`
- Create snapshots with messages, restore to any previous version
- Side-by-side diff comparison between any two commits
- SHA-256 checksums for content integrity verification
- Anime.js-powered timeline UI with staggered animations

### Provenance Manifest System
- AES-256-GCM encrypted origin tokens prove sheet authorship
- Auto-generated on sheet creation and updates
- Admin-only decryption for dispute resolution
- Tamper detection compares current content hash against encrypted manifest
- Architecture ready for scheduled batch tamper scanning

### Feature Flags (OpenFeature-Compatible)
- Database-backed feature flags with admin CRUD
- Percentage-based rollout using deterministic user hashing
- JSON conditions for role-based and user-targeted flags
- In-memory cache with 30-second TTL for performance
- Express middleware (`req.flags.isEnabled()`) for backend gating
- React hook (`useFeatureFlag()`) for frontend gating

### Full-Text Search
- PostgreSQL GIN indexes on sheet title, description, and content
- Combined tsvector index for multi-field ranked search
- `ts_rank` scoring for relevance-ordered results
- Opt-in via `?fts=true` query parameter (gradual migration)
- Indexes also added for User (username) and Course (name, code)

### PWA Offline Support
- Service worker with network-first API caching and cache-first static assets
- Web app manifest for installability (standalone display)
- Apple mobile web app meta tags
- SVG and PNG icon support

### WebAuthn Passkeys (Admin Users)
- FIDO2 passkey registration and authentication for admin accounts
- Built with Node.js crypto — no external WebAuthn dependencies
- Manual CBOR decoder and DER signature verification
- ES256 and RS256 algorithm support
- Passkey management UI in Settings SecurityTab

### Client-Side Image Safety
- Canvas-based pre-upload image safety screening
- File size, dimension, and skin-tone heuristic checks
- Integration hook for future TF.js/NSFWJS model loading
- Non-blocking warnings (server-side OpenAI moderation remains authoritative)

### CI Quality Gates
- GitHub Actions workflow with lint, test, build, accessibility, and bundle size jobs
- Lighthouse CI config with performance (0.8), accessibility (0.9), and Core Web Vitals budgets
- Accessibility config with WCAG 2.2 AA rules and zero-tolerance violations
- 5 MB bundle size budget check

## Enhancements

### Frontend Feature Folder Architecture

- 8 feature barrel exports under `src/features/` (sheets, feed, admin, users, dashboard, notes, auth, settings)
- Each barrel re-exports hooks, constants, and helpers from legacy `pages/` locations
- Convention: new feature logic goes in `src/features/<name>/`, pages import from barrels
- Incremental migration path — no existing imports broken

### Page Decomposition

- FeedPage, SheetsPage, UploadSheetPage rewritten as thin orchestrator shells
- 5 new child components extracted: FeedComposer, FeedAside, SheetsEmptyState, SheetsAside, UploadNavActions
- Average page reduction: -39% lines of code (893 → 452 combined)

### Design Token System Expansion

- 14 new CSS custom property tokens: slate scale (`--sh-slate-50` through `--sh-slate-900`) and info semantics (`--sh-info`, `--sh-info-bg`, `--sh-info-border`, `--sh-info-text`)
- 10 files migrated from hardcoded hex to CSS tokens (~100+ instances)
- All semantic color groups now tokenized: brand, danger, success, warning, info, slate
- Full dark mode support via both light and dark theme token definitions

### HTML Policy Migration (Accept-All Model)

- All HTML is now accepted at submission — no features (scripts, iframes, forms) are auto-blocked
- `validateHtmlForSubmission()` rewritten to structural-only validation (empty/size checks)
- Content flows through detect → classify → route pipeline: Tier 0 (Clean), Tier 1 (Flagged), Tier 2 (High Risk), Tier 3 (Quarantined)
- Tier 1 sheets publish with warning banner and safe preview (scripts disabled)
- Tier 2 sheets route to admin review queue
- Tier 3 sheets quarantined with preview disabled

### HTML Workflow Language Improvements

- "HTML Upload" replaces "HTML Upload Beta" in tutorial modal
- "Safe Preview" replaces "Sanitized Preview" in admin review panel
- Preview description updated to "Full-page preview in a secure sandboxed session"
- "Community guidelines" language replaces "harmful content" framing
- Improved quarantine message with contact support CTA
- Constructive acknowledgement checkbox text

### Sheets Status Filter (Mine View)

- Status filter pills on Sheets page: Drafts, Pending review, Published, Rejected
- Visible only when "Mine" toggle is active
- Backend `GET /api/sheets?mine=1&status=draft` query param support
- Status-aware empty states with custom messaging per filter
- Atomic `toggleMine` callback prevents stale URL param issues

### Feed Avatar Images

- Feed cards now render actual user profile photos (backend returns `avatarUrl` in all author selects)
- Avatar component resolves relative URLs and falls back to initials on image load error
- `onError` fallback applied across all avatar instances: Feed, Profile, Settings, FollowModal

### Upload Hardening
- Magic-byte validation on all file uploads (avatars, sheet attachments, post attachments)
- Triple validation: MIME type + file extension + magic bytes
- ZIP-based Office format detection (DOCX/XLSX/PPTX)
- Automatic file deletion on signature mismatch

### Admin Moderation Dashboard
- Stats overview: total users, new this week, published/draft sheets, feed posts
- Moderation overview: pending cases, active strikes, pending appeals (color-coded)
- Activity log: timeline of recent moderation actions with reviewer info
- Case detail view: full content, confidence score, evidence, quick action buttons
- Sort by date or confidence score

### Performance Optimization
- Web Vitals monitoring (LCP, INP, CLS) with PostHog integration
- Vite manual chunk splitting (react-vendor, animation)
- CSS code splitting enabled
- CLS prevention: min-height on root, aspect-ratio on avatars, contain on card grids
- Resource preconnect hints for Google Fonts and CDNs

### Accessibility
- Skip-to-content link on every page
- `aria-live` route change announcements for screen readers
- ARIA labels on all interactive elements (nav, search, modals, dialogs, buttons)
- `role="dialog"` and `aria-modal` on all modals and dialogs
- `id="main-content"` landmark on all pages
- Global `focus-visible` outline styles
- `prefers-reduced-motion` support (disables all animations)
- Print styles (hide nav, sidebar, toasts)
- Custom scrollbar styles
- `::selection` theming for light and dark modes
- Keyboard shortcut: Ctrl/Cmd+K for quick search

## Security

### Audit Findings & Fixes

- Password reset now enforces the same complexity rules as registration (uppercase + digit required)
- All sensitive endpoints confirmed to have rate limiting
- CSRF protection verified on all state-changing endpoints
- Helmet CSP and security headers confirmed
- File upload triple-validation (MIME + extension + magic bytes)

### Attachment Preview Hardening (2026-03-23)

- All attachment preview iframes use `sandbox="allow-same-origin"` (blocks script execution, popups, forms)
- `referrerPolicy="no-referrer"` prevents origin leakage from preview iframes
- Applied across FeedCard, SheetViewerSidebar, AttachmentPreviewPage

### Error Message Sanitization (2026-03-23)

- All frontend error displays routed through `getApiErrorMessage()` helper
- Prevents raw API implementation details from reaching user-facing UI
- 15 call sites standardized across sheet viewer, feed, and comment hooks

### SECURITY.md
- Vulnerability disclosure policy with response timelines
- Scope definition (in-scope and out-of-scope categories)
- Contact instructions (security@getstudyhub.org)
- Security measures documentation

## Bug Fixes (Cycle 33-34, 2026-03-23)

### P0 — Production Crash

- Feed page crash from `settleSection` calling `.then()` on non-Promise loader returns — wrapped in `Promise.resolve()`

### P1 — Critical

- User profile crash from `optionalAuth` setting truthy `req.user` without `userId` — added `req.user?.userId` guard
- Sheet detail crash from same `optionalAuth`/`userId` issue on starred + reaction lookups
- 14 Prisma validation errors from `parseInt` producing `NaN` — added `Number.isInteger()` guards on all route param parsers

### P2 — Significant

- Feed post reaction delete race condition — `P2025` error not caught on already-deleted row
- Feed avatars never showed images — backend missing `avatarUrl`, frontend component was initials-only
- Broken avatar URLs showed broken image icon — added `onError` fallback across all avatar instances
- SettingsPage silently swallowed initial load failure — now shows error UI with refresh button
- Attachment preview iframes had no sandbox — security gap allowing script execution

### P3 — Minor

- 11 raw `data.error` API strings displayed to users — replaced with `getApiErrorMessage()` helper
- Sheets load-more failure silently swallowed — added toast notification

## Known Limitations

- WebAuthn uses a minimal CBOR/DER implementation; full FIDO2 compliance would benefit from `@simplewebauthn/server`
- TF.js NSFW model is stubbed — current safety check uses canvas heuristics; ML model loading is a future enhancement
- Full-text search uses `?fts=true` opt-in; production migration should switch the default after index verification
- PWA icons are SVG-based placeholders; production icons should be generated as PNGs at 192x192 and 512x512
- Challenge store for WebAuthn uses in-memory Map; production should use Redis or database-backed sessions
